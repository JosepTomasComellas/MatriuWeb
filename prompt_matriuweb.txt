Vull que actuïs com un arquitecte i desenvolupador sènior especialitzat en Blazor (.NET 10), Docker, Docker Compose, Nginx, SSL/TLS, Redis, PWA, Grafana, Prometheus i desplegament en Proxmox LXC.

Necessito que em generis una solució completa, funcional, executable i estructurada com un repositori GitHub real, llesta per desplegar en un entorn productiu lleuger.

# OBJECTIU DEL PROJECTE

Crear una aplicació web en C# amb Blazor (.NET 10) que funcioni dins de Docker, desplegada amb docker compose, amb:

- Nginx com a reverse proxy HTTPS
- Redis com a cache intern
- Persistència en fitxer JSON
- PWA instal·lable
- Pensada per funcionar en una LXC de Proxmox

La web serà un dashboard de monitorització amb una matriu configurable d’iframes per mostrar eines de monitorització i altres webs internes.

A més, vull que la solució inclogui també tota la infraestructura necessària per monitoritzar l’estat de l’entorn amb Grafana, Prometheus i exportadors/agents adequats, de manera que aquests dashboards també es puguin visualitzar des de la pròpia MatriuWeb.

# ENTORN REAL

- Hostname intern de l’LXC principal: ct-matriuweb.tute.int
- URL pública final: https://tuteapps.ddns.net:4444
- Publicació via:
  - Nginx

  - Port 4444

# REQUISITS FUNCIONALS DE MATRIUWEB

## Dashboard principal
- La pàgina principal ha de mostrar una matriu configurable d’iframes
- Nombre mínim de frames: 2
- El nombre de frames ha de ser configurable des de la UI
- El layout de la graella s’ha d’adaptar automàticament al nombre de frames
- Cada frame ha de tenir:
  - títol
  - URL
  - refresh configurable
  - estat visual
  - opció d’obrir en nova pestanya

## Vista enfocada
- S’ha de poder seleccionar un únic frame
- La selecció s’ha de fer des d’un botó de la barra superior
- Quan hi ha un frame seleccionat:
  - el frame seleccionat es mostra gran
  - la resta es mostren petits en zona secundària
- Ha d’existir un botó per tornar a la vista matriu

## Configuració des de la UI
Cal una pantalla de configuració que permeti:
- afegir frames
- editar frames
- eliminar frames mantenint sempre un mínim de 2
- reordenar frames
  - prioritat a botons pujar/baixar
  - drag & drop només si queda net i mantenible
- configurar el refresh global i/o per frame
- activar o desactivar frames
- definir quin perfil és el perfil actiu

## Perfils
Vull suport per múltiples perfils de configuració:
- crear perfil
- duplicar perfil
- editar perfil
- eliminar perfil
- seleccionar perfil actiu
- cada perfil ha de poder tenir:
  - frames propis
  - ordre propi
  - refresc propi
  - vista per defecte
  - opcions kiosk pròpies si té sentit

## Importació / exportació
- exportar configuració a JSON
- importar configuració des de JSON
- validar abans d’aplicar
- conservar còpia de seguretat abans de substituir

## Backups
- carpeta de backups:
  /docker/MatriuWeb/data/config/backups/
- guardar versions amb timestamp
- conservar els últims N backups
- permetre restaurar una configuració des de la UI si és viable
- si no és viable des de la UI, com a mínim amb script

## Validació
- mínim 2 frames
- títol obligatori
- URL ben formada
- ordre coherent
- refresh en valors raonables
- el sistema ha de ser robust davant JSON corrupte

## Inicialització robusta
- si no existeix el JSON, crear-lo amb configuració mínima vàlida
- si el JSON està corrupte, intentar restaurar l’últim backup vàlid
- si hi ha menys de 2 frames, regenerar una configuració mínima segura o rebutjar la configuració i avisar

# UX, RENDIMENT I GESTIÓ D’ERRORS

## Auto-refresh
- suport global i per frame
- valors típics: 10s, 30s, 60s, 120s
- possibilitat de pausar el refresh
- si és viable, pausar el refresh quan la pestanya no està activa
- pausar o reduir refresh en mode configuració si té sentit

## Loading state
- skeleton o placeholder mentre carrega
- indicador visual de càrrega
- timeout visible si és viable

## Errors i fallback d’iframe
Cal gestionar correctament:
- URL invàlida
- timeout
- iframe bloquejat per X-Frame-Options
- iframe bloquejat per Content-Security-Policy
- errors de càrrega
- errors de connectivitat

Quan això passi, el frame ha de mostrar:
- estat visual clar
- missatge d’error entenedor
- botó “Obrir en nova pestanya”

## Estat de salut dels frames
Afegeix un petit indicador visual per frame:
- OK
- lent
- inaccessible
- bloquejat
- carregant
